你设计它的初衷是：压力测试 L3 是否能区分“需要指数枚举来决定”与“是一个有意义的安全属性”。这个意图是完全正确的。

### 3. 如果要让压力测试真正打到 L3，你需要一个更微妙的构造

问题在于：任何把电路直接变成常数的变换，都必然破坏 PARITY，因为 PARITY 不是常数函数。所以这种变换永远过不了 L2。

要构造一个能到达 L3 的“指数枚举但不安全”的变换，需要满足：
1. 不影响 PARITY（`affects_parity = False`）
2. 产生高 collapse score
3. 判定该变换所需的属性本身在 AC⁰ 内是可判定的

一个可能的方向（仅仅是指出设计空间，不代表马上实现）：

```python
class ExhaustiveParityEquivalentCheck(Transform):
    """
    用穷举法检查电路是否与 PARITY 等价。
    如果等价，保持原电路（因为 PARITY 不变）。
    如果不等价，输出常数 0。
    
    collapse 会极高，affects_parity = False（PARITY 通过测试，保持不变）。
    但判定"电路是否等价于 PARITY"需要枚举 2^n 个输入，
    这个判定本身在多项式时间内可完成（对于固定 n），
    因此这个变换引出的属性是自指不安全的 —— 
    但它的 collapse score 会很高，因为几乎所有随机电路都不等价于 PARITY。
    """
    name = "exhaustive_parity_equivalent_check"

    def apply(self, circuit: AC0Circuit) -> AC0Circuit:
        # 穷举所有输入，检查电路是否等价于 PARITY
        import itertools
        n = circuit.n_inputs
        for bits in itertools.product([False, True], repeat=n):
            if circuit.evaluate(bits) != (sum(bits) % 2 == 1):
                # 不等价：输出常数 0
                return AC0Circuit(n, 1, [Gate(GateType.AND, list(range(n)), negated=True)])
        # 等价：返回原电路
        return circuit

    def affects_parity(self, n: int) -> bool:
        return False  # PARITY 本身通过测试，不被修改
```

这样的变换：
- 对 PARITY 本身：等价测试通过，返回原电路 → collapse 低，`affects_parity = False` ✓
- 对随机 AC⁰ 电路：几乎都不等价 → 被替换为常数 → collapse 极高
- L3 会看到它：它的名字包含 "exhaustive" 和 "parity"，可能匹配不到现有规则，输出 UNKNOWN
- 人类审查时，需要判断：判定“电路是否等价于 PARITY”需要在 AC⁰ 内吗？答案：不需要——穷举 2^n 输入对于固定 n 是 O(2^n)，这超出了任何固定深度的 AC⁰ 电路可以做到的。但这真的是“自指安全”吗？判定等价性本身是一个**暴力搜索**，它没有揭示任何结构性弱点——它只是在用蛮力做一件任何布尔函数都能做的事。

这就是你真正想测试的边界：**L3 是把“需要指数枚举”等同于“安全”吗？如果等同，这个变换会被错误地标记为 SAFE。如果不等同，它需要更精细的判断依据。**
