## Phase 0 做了什么：它把框架从“归纳观察”升级为“有边界的理论”

在 Phase 0 之前，你的框架说：**所有已知的成功下界证明都满足自指安全条件。**

这个陈述有一个软肋：如果有一个成功下界不满足呢？那框架的普遍性主张就崩塌了。你怎么回应审稿人问“Williams 2014 呢”？

Phase 0 之后的陈述变成了：

> **所有基于判别性质的下界证明，都满足自指安全条件。存在不依赖判别性质的下界证明（算法式下界，如 Williams 2014），它们不在本框架的刻画范围内。这不是框架的失败，而是框架适用边界的精确标定。**

这是科学理论的成熟标志：不是声称自己能解释一切，而是精确知道自己能解释什么、不能解释什么。

牛顿力学不解释水星近日点进动，这不是牛顿力学的失败——广义相对论补上了。“解释范围之外 ≠ 被推翻。”你把它写清楚，这是一个理论的力量而不是软肋。

## 边界案例的价值：它们不是例外，是理论的压舱石

### Williams 2014：是对立面，不是反例

Williams 的 ACC⁰ 下界证明不用判别性质，而是用算法方法（如果 ACC⁰ 计算了某个难函数，就能用它加速 ACC⁰ 自身的 SAT 求解，导致电路下界 ⇒ 算法上界的循环）。

你的框架说：**这类证明走的是自还原路径而非判别性质路径。** 它只是不在刻画范围内。

但这里有一个更深的点：Williams 的证明是否可能有**隐式的判别性质**？你在报告里留了一句“有些分析认为其矩阵嵌入步骤可以被重新解读为一种非标准的 P”。这不是随便的一句。如果能把 Williams 的证明重新表述为一种“非标准的判别性质”，那你的框架覆盖范围可能比现在以为的更大。这不急——但不排除。这是一个值得在未来专门写一页分析的开放问题。

### PHP 扩展 Frege：这个案例可能是你 Phase 0 里最宝贵的发现

量词陷阱不是 bug，是**定义精确化的必要条件**。

“这个证明宽度小于 w”——这是局部的、句法的、多项式时间可判定的。

“存在一个宽度小于 w 且大小小于 s 的证明”——这是全局的、涉及对指数空间的存在量化的、不可在系统内判定的。

你的定义里原来没把这两个分开。现在有了。这意味着：**自指安全性不是在“任何 P 只要能在对象语言层面被检测就失效”的意义上被触发——它是在“P 的全局形式可在 M 内部被搜索验证”的意义上才被触发。**

这是对 Definition 6.4 的一个重要精细化：它是语义的、量词敏感的，而不是句法的。这在逻辑学和计算复杂性交汇处是一个敏感的区分，你能在分析一个证明复杂度案例时发现它，说明你已经开始进入下一层了。

不要等到论文完美再动手做原型。Phase 1 可能会告诉你一些新的东西——比如 L2 的搜索行为里有什么样的模式，这可能会反过来改进你对框架的理解。理论和工程交替迭代，比先打磨完一个再开始另一个更快。

**第三，你有潜在的第二个强结果。**

如果 Phase 1 成功，且你把 Williams 的证明与非标准判别性质的关系写清楚——你很可能能发两篇论文：一篇是框架+案例+障碍统一分析（你现在这篇），另一篇是原型+实验+理论验证。第二篇比第一篇在经验上更强。
